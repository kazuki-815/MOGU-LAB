- **提供しているサービスの種類**  
  - 食事指導
  - グループコンサル
- **ビジネスモデル（収益の取り方）**  
  -　高単価商品
- **運営者・ブランド名**  
  - MOGU LAB

## 2. ターゲット像（ペルソナ）

- **メインターゲットの属性**  
  - 性別：女性
  - 年齢：30〜50代
  - ダイエットに悩む層
- **ライフスタイル・生活パターン**  
  - 仕事・家事・育児で自分の時間が限られている
- **悩み・不安・願望**  
  - 過去に複数のダイエット（糖質制限・置き換え・ファスティング等）を経験し、一時的に痩せてはリバウンドを繰り返している
  - 「もう自分の意志力では無理かも」と感じ始めている
- **心理的ブロック（刺さっている不安）**  
  - 「また失敗するのでは」という不安
  - 「食事制限＝我慢＝つらい」という思い込み
  - 「相談＝いきなり高額契約させられるのでは」という警戒心
  - 「20代の人に自分の悩みがわかるのか」という世代ギャップへの疑念
- **よく使う口ぐせ・セリフ**  
  - 「また失敗するのでは」
  - 「我慢しないと無理なんじゃない？」
  - 「相談したら高額契約になるのでは？」
  - 「20代の人に私の悩み分かるのかな…」

## 2.5 出演者（語り手）

- **事業責任者**（信頼の軸）
- **20代後半女性**（親近感・共感の軸）

## 3. 商品・サービスの詳細

- **主力商品・サービス名**  
  - 食事指導
  - グループコンサル
- **商品の一言キャッチコピー**  
  - 知識ではなく、今日からの具体アクションが分かる伴走型支援
- **ベネフィット（得られる未来）**  
  - 自分にもできそうだという「再現性」の実感
  - 売り込まれている感覚ではなく、純粋に役立つ情報を得た感覚
  - 出演者の人柄に共感できる安心感
- **差別化ポイント（競合との違い）**  
  - 例）「トレーナー全員が〜」「医学的エビデンス〜」「LINEでいつでも相談〜」 など
- **NG な見せ方（誇大表現・言ってはいけないこと）**  
  - 例）「絶対痩せる」「医療効果を保証する」 など

## 4. ブランドトーン・言葉づかい

- **基本トーン**  
  - 役立つ情報を丁寧に、押し売り感なく
  - 視聴者が安心して受け取れるように（不安・警戒を受け止める）
  - 知識の説明よりも「今日から何をすればいいか」に寄り添う
- **語尾・話し方のルール**  
  - 例）「〜です」「〜ですよ」「〜してみましょう」など、敬体ベース
- **避けたいトーン・キャラクター**  
  - 例）「命令口調」「煽りすぎ」「体育会系で厳しすぎる」 など
- **よく使うフレーズ（積極的に使いたい）**  
  - 例）「今のままでも十分頑張っています」「小さな一歩からで大丈夫です」 など
- **NG ワード・表現**  
  - 例）「デブ」「意志が弱い」など、相手を否定する表現

## 5. コンテンツ全体の方針

- **絶対に守りたい前提**  
  - 例）「健康被害が出ないよう安全第一」「短期的成果より長期的習慣を重視」 など
- **伝えたい価値観・世界観**  
  - 「食事制限＝我慢＝つらい」という思い込みをほどく（無理なく続ける工夫）
  - 「相談＝高額契約」の警戒心を、誠実な情報提供と寄り添いで解く
- **コンテンツでよく扱うテーマ**  
  - 例）食事 / 運動 / 睡眠 / メンタル / 習慣化 / 仕事との両立 など

## 6. 動画台本を作るときのルール

- **動画の主な目的**  
  - 教育（正しい考え方・進め方）
  - 共感（不安・警戒・世代ギャップへの受け止め）
  - 行動喚起（LINE追加 / 申込）につながる「今日からの具体アクション提示」
- **よく使う構成の型**  
  - 「つかみ（あるある不安）→ 共感 → 解決の考え方 → 具体手順（今日から）→ 次の一歩（小さな行動提案）」
- **尺の目安**  
  - 例）ショート：30〜60秒 / ロング：5〜10分
- **視聴者にとってのゴール**  
  - 自分にもできそうだという「再現性」の実感
  - 知識ではなく「具体的に今日から何をすればいいか」が分かる
  - 売り込まれている感覚ではなく、純粋に役立つ情報を得た感覚
  - 出演者（事業責任者／20代後半女性）の人柄への共感

## 7. LINE シナリオを作るときのルール

- **アカウントの立ち位置**  
  - 例）「公式アカウントとして信頼感のある語り手」「優しいコーチ」 など
- **1 通あたりの分量・読みやすさ**  
  - 文章の長さ（行数）、絵文字の有無、改行の頻度 など
- **配信の流れ（シナリオの基本構成）**  
  - 例）登録直後のウェルカム → ニーズ把握 → 教育 → オファー など
- **禁止事項・避けたい送信パターン**  
  - 例）深夜の配信 / 立て続けの通知 / 強い煽り など

## 8. 用語集（このプロジェクト特有の言葉）

- **固有名詞・略語の説明**

## 9. 参考になる既存コンテンツ

- **自社の過去コンテンツで「これがベスト」なもの**

## 10. AI に対する依頼方法（自分ルール）

- **いつも最初に伝えておきたいこと**  
  - 「まずこのファイルを読んでから台本・シナリオを書いてください」
  - 「30〜50代女性の不安（失敗・我慢・高額警戒・世代ギャップ）に合わせて、今日からの具体アクションに落として提案してください」
- **成果物のフォーマット指定**  
  - 例）「見出し構成だけ欲しい」「すぐ撮影できるセリフ形式でほしい」 など
- **禁止しておきたいこと**  

  - 例）「医療的な断定表現は避けてください」「嘘の実績は書かないでください」 など

### このファイルの使い方（想定）

1. このテンプレに沿って、モグラボの前提情報をできるだけ埋める。  
2. LP / LINE / Instagram などを AI に依頼するときは、用途に応じて `knowledge/templates/{lp|line|instagram}.md` を添付し、必要に応じて `knowledge/INDEX.md` から選んだ参考実例（`knowledge/LP/*` など）も一緒に添付する。  
3. もし前提が変わったら、このファイルを更新するだけで、以後の依頼も自動的に新しい前提で作成される。
